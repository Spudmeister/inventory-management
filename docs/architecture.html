Request Data Flow

Filter → API → Response

1 — User

FilterBar.vue

Selects Period, Warehouse, Category, or Status from sticky dropdowns.

2 — State

useFilters.js

Singleton composable holds four reactive refs shared across all views.

3 — View

View Component

Watches filter changes, calls getCurrentFilters() and passes object to API client.

4 — Client

api.js

Builds query string (skips "all" values), issues axios.get() to backend.

5 — Server

FastAPI

apply_filters() + filter_by_month() filter in-memory lists. Pydantic validates response.

6 — Render

Vue Template

Stores raw data in refs; computed properties derive display values and aggregates.

selectedLocation = "Tokyo" → getCurrentFilters() → GET /api/orders?warehouse=Tokyo&month=2025-09 → apply_filters(orders, "Tokyo") → filter_by_month(..., "2025-09") → List[Order] JSON

State Management (Composables)

No Vuex or Pinia. Singleton composables serve as shared reactive state.

  • selectedPeriod

  • resetFilters()
  • hasActiveFilters

  • locale-aware name/title
  • mock tasks list

Key Design Decisions

Frontend Patterns

ReactivityRaw data in refs; derived values in computed properties
ChartsCustom SVG — no external chart library
ModalsProps down / events up; API called by parent
v-for keysAlways unique IDs (sku, order.id), never index
i18nEN/JA locale; USD/JPY currency auto-bound

Backend Patterns

DataJSON loaded once at startup into module-level variables
FilteringGET query params only — no POST for reads
QuartersQ1-2025 → ["2025-01","2025-02","2025-03"] via QUARTER_MAP
InventoryNo time dimension — month filter does not apply
Tasks/POsIn-memory only; resets on server restart
Ports & Services

Frontend — :3000

Vite dev server. Hot module replacement. Proxies nothing — uses absolute localhost:8001 URLs.

Backend API — :8001

Uvicorn + FastAPI. CORS allow-all for development. Interactive docs at /docs.

No Database

All data lives in 7 JSON files. Tasks and purchase orders stored in memory and reset on restart.
